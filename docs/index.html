Add Item to Empty List

For instance, in an empty list, create an item named "Buy new John Grisham book"

Before:

This is what an empty list looks like in our app. To create an item, the user can click on the plus icon in the bottom right corner of the screen. This will pull up a page to create a new list, shown below.

Note that if you click on the plus icon, the button will change color and compress slightly, indicating that it has been activated.

During:

Next, the user is brought to a page to create a new list item. They will be able to change the name of the task (for instance, they can name it "Buy new John Grisham book"), the due date, and the location. They can also add any relevant notes. If the user decides they don't want to create the task, they can press the X in the top left corner, which cancels the task creation. When the user is done creating a task, they can press the check in the upper right corner to save the task to the list.

The X and check buttons respond to being pressed by changing size and color. Currently, only the name field is editable on this page. We plan to implement the rest of this form when we transfer our app over to JavaScript / React.

After:

When the user presses the checkbox, confirming the creation of the new task, they are brought back to the single list page, where the task and its due date are visible. Note that the completed box is, by default, left unchecked.

Add Item to Non-Empty List

For instance, in a non-empty list, create an item named "Eat Lunch"

Before:

Notice that our Daily Tasks list is not currently empty. To create an additional item in this list, the user can press the plus icon in the bottom right of the screen. This is the same button the user would use to create an item in an empty list. As before, when the user clicks the plus icon, they will be brought to a page to create a new task.

During:

The user is brought to the same screen that we saw earlier. To cancel the creation of this new task, the user can press the X button. To confirm the creation of the task, they can press the check button. They can edit fields like date, time, location, etc.

After:

The list item the user created ("Eat lunch") has been added to the existing list.

Mark List Item Completed

For instance, mark the item named "Call Mom" completed (assumes there exists an item named "Call Mom").

Before:

As before, this is the single list page where the user can view all the tasks in a given list, and also create new tasks. All features are the same as in the walkthrough of the adding an item to an empty list task.

During:

To mark an item completed, the user can simply tap the checkbox next to the item they want to mark completed. There is no intermediate screen.

After:

Mark List Item Completed (Alternate Method)

For instance, mark the item named "Call Mom" completed (assumes there exists an item named "Call Mom").

Before:

As before, this is the single list page where the user can view all the tasks in a given list, and also create new tasks. All features are the same as in the walkthrough of the adding an item to an empty list task.


When the user taps on the "Call Mom" task on the single list page shown above, they are brought to this page which allows them to view task details. The user can then tap on the pencil icon to edit the task.


Once the user is on the edit task page, they can check the "Completed" checkbox to mark that task as completed.

During:

To mark the item completed, the user can simply tap the "Completed" checkbox on the edit task page. There is no intermediate screen.

After:

After the user taps the checkbox, the task is marked completed (indicated by the filled-in checkbox). The user can tap the check button in the top right corner to save the change they have made to this task's completion status.


After the user saves their change to the task's completion status, the list item is marked completed on the single list page (indicated by the filled-in checkbox).

Rename/Edit List Items

For instance, rename the item "Text John" to "Text John about bank statements" (assumes there exists an item named "Text John").

Before:

From the list page, the user can tap the pencil icon next to the list item they wish to rename/edit. This takes them to a page to edit their list item.


During:

From this edit task page, the user can click on the task name and rename it to their desired new name.

After:

Afterward, the user can press the checkmark to save their changes to the task name, and the name of their list item will be updated on the main list page. Once the user presses the checkmark, they will automatically be returned to the main list page.

Create a new list

Before:

If the user would like to create a new list from the Home Page, they can press the plus icon in the bottom right of the screen. Currently, this button will change color and compress slightly when clicked. Pressing this button will bring the user to the Create New List page. Notice that, before the user types a name for their list, they cannot create a new list -- the checkmark is greyed out.


During:

In the Create New List page, the user can type the name of their list and choose an icon for their list. The icon chosen will be shown in teal. Once the user types a name for their list, the check icon will turn white again. If the user wants to cancel the creation of a new list, they can press the X button. If they want to finish creating their list, they can press the checkmark. In both cases, the button pressed will compress and change color slightly to show it has been activated.

After:

Once the checkmark is pressed, the user is brought back to the home page. Notice that their newly-created list has been added to the top of the page, with their chosen name and icon.

Back to top of page

Edit the appearance of a list

Before:

Users may want to change a list's name or icon. To do so, the user can swipe left on a list from the Home Page. This will reveal two options: editing the list (indicated by a pencil) and deleting the list (indicated by a trashcan). To edit the list, the user can press the pencil. This will pull up the Edit List Page.



During:

From the Edit List Page, the user can change the name or icon of the list. They can also delete the list by pressing the red trashcan bar at the bottom of the screen. In this instance, the user will change both the name and the icon of their list. At any time, the user can confirm their changes by pressing the checkmark or cancel their changes by pressing the X.

After:

Once the checkmark is pressed, the user is brought back to the home page. Notice that the user's edits to their list have been made.

Back to top of page

Delete a list

Before:

One way to delete a list is through the home screen. The user can swipe left on any list. This will bring up the options to edit or delete the list.

During:

The user can press the red trashcan. This will delete the list (we plan to add warnings before deleting lists).

After:

The list and its contents have been removed.

Back to top of page

Delete a list (Alternate Method)

Before:

Another way to delete a list is through the Edit List screen. The user can swipe left on any list. This will bring up the options to edit or delete the list. Then, they can press the pencil to edit their list.


During:

The user can then press the red trashcan bar at the bottom of the screen. This will delete the list (as stated before, we plan to add warnings before deleting lists).

After:

The list and its contents have been removed.

Search for a list

Before:

To search for a list, the user can press the magnifying glass in the top right corner. This will bring up the search screen.


During:

The user can then type to search for their list and filter out other lists. At any point, they can cancel their search by pressing "Cancel." In this case, suppose the user was looking for the "Birthday Gifts" list. They can then click on that list from the search menu and the list will open.

After:

The list has been found through searching and is now open.
