CS 124 - Lab 1 Walkthrough

Add Item to Empty List

For instance, in an empty list, create an item named "Buy new John Grisham book"

Before:

This is what an empty list looks like in our app. To create an item, the user can click on the plus icon in the bottom right of the screen. This will pull up a page to create a new list, shown below.

Note that if you click on the plus icon, the button will change color and compress slightly, indicating that it has been activated.

During:

Next, the user is brought to a page to create a new list item. They will be able to change the name of the task (for instance, they can name it "Buy new John Grisham book"), the due date, the location, and add any relevant notes. If the user decides they don't want to create the task, they can press the X in the top left corner, which cancels the task creation. When the user is done creating a task, they can press the check in the upper rght corner.

The X and check buttons respond to being pressed by changing size and color. Currently, only the name field is editable on this page.

After:

When the user presses the checkbox, confirming the creation of the new task, they are brought back to the list page, where the task and its due date are visible. Note that the completed box is, by default, unchecked.

Add Item to Non-Empty List

For instance, in a non-empty list, create an item named "Eat Lunch"

Before:

Notice that our list is not currently empty. To create an additional item in this list, the user can press the plus icon in the bottom right of the screen. This is the same button the user would use to create an item in an empty list. As before, when the user clicks this icon, they will be brought to a page to create a new task.

During:

The user is brought to the same screen that we saw earlier. To cancel the creation of this new task, the user can press the X button. To confirm the creation of the task, they can press the check. They can edit fields like date, time, location, etc.

After:

The list item the user created has been added to the list.

Mark List Item Completed

For instance, mark the item named "Call Mom" completed (assumes there exists an item named "Call Mom").

Before:

As before, the user can create the task in this screen. All features are the same as they were when the user added a task to an empty list.

During:

To mark an item completed, the user can simply tap the checkbox next to the item they want to mark completed. There is no intermediate screen.

After:

After the user taps the checkbox, the list item is marked completed (indicated by the filled-in checkbox).

Rename/Edit List Items

For instance, rename the item "Text John" to "Text John about bank statements" (assumes there exists an item named "Text John").

Before:

From the list page, the user can press on the pencil icon next to their list item. This takes them to a page to edit their list item.

During:

From this edit page, the user can click on the task name and rename it to their desired new name.

After:

Afterwards, the user can press the checkmark, and the name of their list item will have changed on the main list page.

Show only uncompleted items.

Before:

Before, we can see that completed items still appear on the list page. To hide list items, the user first presses the three dots in the top right corner.

During:

A menu appears with options to edit the list and change which items are shown. To show only uncompleted items, the user can press the button that says "Hide Completed Tasks."

After:

Now, only uncompleted tasks are shown.

Delete all completed items.

Before:

This process is similar to hiding completed items. However, in this process, you will not be able to bring back completed items after you delete them. Before deleting completed items, we can see that completed items still appear on the list page. To delete these completed items, the user will presses the three dots in the top right corner. This will bring up the menu we saw earlier.

During:

A menu appears with options to edit the list and change which items are shown. To delete completed items, the user can press the button that says "Delete completed tasks."

After:

Now, only uncompleted tasks remain.
